# 朝堂治理系统一期总纲
性质：奏折
日期：2026-03-21
提交：中书省

## 一、事项名称
朝堂治理系统一期建设总纲。

## 二、目的与结论
此系统一期之目标，不在制作展示性看板，而在建立可立项、可流转、可监控、可统计、可留痕、可归档之治理底座。

结论如下：

一期建设应以“制度先行、边界先定、闭环优先”为原则，先完成项目治理最小可用闭环，再循序扩展自动化、复杂审批、深度协同与高级报表能力。

## 三、范围边界
### （一）一期纳入范围
1. 项目立项与统一编号
2. 项目列表与检索
3. 项目详情页
4. 基础流程流转
5. 朝堂总览页
6. 国库统计基础版
7. 审计日志页
8. 基础告警能力
9. 基础角色权限控制

### （二）一期暂不纳入范围
1. 复杂自定义工作流编排
2. 全自动 Agent 治理
3. 高级报表中心
4. 甘特图与复杂时间轴
5. 深度外部系统集成
6. 纯装饰性展示能力

## 四、建设原则
1. 先治理，后美化
2. 先标准化，后灵活化
3. 先最小可用，后逐步扩展
4. 先留痕，再自动化
5. 先分层，再统览
6. 先定边界，再谈效率

## 五、一期核心模块
### （一）朝堂总览
用于展示全局态势，至少包括：
- 项目总数
- 进行中项目数
- 已完成项目数
- 延期项目数
- 高风险项目数
- 待决事项数
- 最近告警
- 国库消耗概况
- 各部门负载概览

### （二）项目台账
对全部项目统一登记与检索，至少包含：
- 项目编号
- 项目名称
- 项目目标
- 负责人
- 协同部门
- 当前阶段
- 当前状态
- 优先级
- 风险等级
- 创建时间
- 截止时间
- 预算
- 已消耗
- 最新进展
- 下一步动作

### （三）流程监控
围绕事务生命周期建立标准阶段：
- 提案
- 草拟
- 审议
- 批准
- 执行
- 验收
- 归档

并记录每一步之处理人、开始时间、处理时间、结果、批注、是否超时、是否阻塞。

### （四）国库统计
统一统计：
- token 消耗
- 项目预算
- 实际支出
- 各部门占用
- 日／周／月趋势
- 阈值预警
- 超支记录

### （五）告警监察
一期重点告警包括：
- 服务异常
- 接口失败
- 数据异常
- 项目延期
- 长时间无推进
- 超预算
- 流程卡滞
- 风险事项未处理
- 权限异常

### （六）审计与归档
所有关键行为必须留痕，包括：
- 立项
- 状态变更
- 流程推进
- 审批结果
- 预算调整
- 权限变更
- 归档／反归档

## 六、制度口径
### （一）阶段口径
统一阶段为：提案、草拟、审议、批准、执行、验收、归档。

### （二）状态口径
统一状态为：未开始、进行中、阻塞中、已延期、已完成、已归档、已终止。

### （三）优先级口径
统一优先级为：P0、P1、P2、P3。

### （四）风险等级口径
统一风险等级为：低、中、高、极高。

## 七、角色分工
### 陛下
观全局、定方向、批关键事项、看重点告警。

### 宰相
统筹轻重先后、调整优先级、协调跨项目事项。

### 中书省
主草拟系统结构、页面逻辑、栏目方案、字段命名与制度表达。

### 门下省
主审流程制度、权限边界、审议规则、告警条件、合规留痕。

### 尚书省
主实施路径、技术架构、研发排期、上线节奏与交付落地。

### 承办角色
在授权范围内承接执行任务并更新进度。

## 八、推进路径
### 第一阶段：立规定边界（1周）
交付物：系统模块清单、页面结构清单、字段初稿、权限矩阵、一阶段范围确认、风险与依赖清单。

### 第二阶段：原型与数据模型（1至2周）
交付物：原型图、数据模型草案、API 草案、状态流转图、首页总览初稿。

### 第三阶段：一期开发（2至4周）
交付物：登录与角色权限、总览页、项目列表页、项目详情页、基础流程流转、国库统计页、审计日志页。

### 第四阶段：联调试运行（1至2周）
交付物：测试问题清单、权限修订、告警规则初版、稳定性修补、试运行反馈报告。

### 第五阶段：二期增强
扩展方向：更复杂审批流、甘特图／时间轴、自动提醒与催办、更细国库报表、健康度评分、Agent 协同接入、自动周报／月报。

## 九、一期验收标准
一期验收至少应满足：
1. 能建立项目
2. 能查看项目列表
3. 能进入详情
4. 能推进流程
5. 能标记风险与状态
6. 能统计基础资源消耗
7. 能查看关键审计记录
8. 能完成项目归档

## 十、风险与约束
1. 一期范围过大，致使迟迟不能落地
2. 状态定义不统一，导致统计失真
3. 权限边界不清，导致越权操作
4. 审计不完整，导致事后无法追责
5. 告警过多过杂，削弱治理效果
6. 只重界面展示，不重流程闭环
7. 数据口径不一致，导致国库统计失真

## 十二、结语
臣等以为，此系统应定位为朝堂治理底座，而非普通展示看板。其建设顺序，当遵“先定制度与边界，再定结构与字段，再成一期闭环，后续逐步扩展”之序。如此，则朝堂诸务，方可由散乱归于秩序，由经验归于制度，由被动应付归于可视可管。

# 一期-状态流转图
性质：附件
日期：2026-03-21
提交：中书省

## 一、标准阶段
提案 → 草拟 → 审议 → 批准 → 执行 → 验收 → 归档

## 二、阶段说明
### 1. 提案
提出项目设想，形成立项意向。

### 2. 草拟
补齐目标、范围、负责人、预算、风险与下一步安排。

### 3. 审议
对事项必要性、边界、权限、预算与风险进行审看。

### 4. 批准
对可实施事项作正式准许或退回处理。

### 5. 执行
依批准内容推进实施，并持续更新进展。

### 6. 验收
对交付结果、目标完成度、资源消耗与遗留问题进行确认。

### 7. 归档
冻结状态，保留完整留痕，进入结案查询。

## 三、主流转规则
1. 提案可进入草拟
2. 草拟可进入审议
3. 审议可进入批准
4. 批准通过后进入执行
5. 执行完成后进入验收
6. 验收通过后进入归档

## 四、退回规则
1. 草拟不充分，可由审议节点退回草拟
2. 审议不通过，可退回草拟或终止
3. 批准不通过，可退回审议或终止
4. 验收不通过，可退回执行继续整改

## 五、状态规则
### 1. 状态枚举
- 未开始
- 进行中
- 阻塞中
- 已延期
- 已完成
- 已归档
- 已终止

### 2. 状态使用说明
1. 项目进入任一有效办理阶段后，状态应转为进行中
2. 因依赖、权限、资源、决策原因无法推进时，可标记为阻塞中
3. 超过截止日期未完成时，可标记为已延期
4. 验收通过后，可标记为已完成
5. 归档完成后，状态应转为已归档
6. 明确停止推进时，可标记为已终止

## 六、阻塞与升级规则
1. 任一阶段长时间无推进，应生成卡滞告警
2. 阻塞超过阈值时，应升级至宰相处理
3. 涉及重大预算、权限、方向调整时，应上达宰相或陛下

## 七、留痕要求
每次阶段推进、退回、阻塞、终止、归档，均应记录：
1. 操作人
2. 操作时间
3. 原阶段与新阶段
4. 原状态与新状态
5. 原因说明
6. 批注内容

## 八、文字图示
提案
  ↓
草拟
  ↓
审议
  ├─ 不通过 → 退回草拟
  ↓
批准
  ├─ 不通过 → 退回审议／终止
  ↓

  ├─ 阻塞 → 保持执行阶段并标记阻塞中
  ↓
验收
  ├─ 不通过 → 退回执行
  ↓
归档

# 实施拆解与开工清单

## 一、一期最小可用闭环

建议先落以下 7 件事：

1. 统一枚举与口径
   - 阶段
   - 状态
   - 优先级
   - 风险等级
2. 建立核心数据模型
   - projects
   - workflow_records
   - approvals
   - alerts
   - audit_logs
3. 明确角色权限校验
   - 立项
   - 审议
   - 批准
   - 执行进展更新
   - 归档
4. 完成 4 个核心页面
   - 总览页
   - 项目列表页
   - 项目详情页
   - 审计日志页
5. 接通基础流程流转
   - 提案 → 草拟 → 审议 → 批准 → 执行 → 验收 → 归档
6. 建立基础告警
   - 延期
   - 卡滞
   - 超预算
   - 权限异常
7. 做好审计留痕
   - 状态变更
   - 流程推进
   - 审批结果
   - 权限变更
   - 预算调整

## 二、研发排期建议

### 第 1 周
- 锁定范围边界
- 定稿字段字典
- 定稿权限矩阵
- 定稿状态流转规则

### 第 2 周
- 出数据库表结构
- 出 API 草案
- 出页面原型图
- 明确告警阈值

### 第 3-4 周
- 开发登录与 RBAC
- 开发总览页
- 开发项目列表与详情页
- 开发流程流转
- 开发审计日志

### 第 5 周
- 联调测试
- 权限修订
- 告警规则校准
- 试运行反馈

# 一期-字段字典初稿
性质：附件
日期：2026-03-21
提交：中书省

## 一、项目主表字段
| 字段名 | 含义 | 类型 | 必填 | 备注 |
| --- | --- | --- | --- | --- |
| id | 主键标识 | string | 是 | 系统唯一标识 |
| code | 项目编号 | string | 是 | 按统一规则生成 |
| name | 项目名称 | string | 是 | 正式名称 |
| goal | 项目目标 | text | 是 | 项目要达成之事 |
| description | 项目描述 | text | 否 | 补充说明 |
| sponsor | 发起人 | string | 是 | 提案或立项人 |
| owner | 负责人 | string | 是 | 主责人 |
| departments | 协同部门 | array | 否 | 多部门可并列 |
| stage | 当前阶段 | enum | 是 | 提案/草拟/审议/批准/执行/验收/归档 |
| status | 当前状态 | enum | 是 | 未开始/进行中/阻塞中/已延期/已完成/已归档/已终止 |
| priority | 优先级 | enum | 是 | P0/P1/P2/P3 |
| riskLevel | 风险等级 | enum | 是 | 低/中/高/极高 |
| createdAt | 创建时间 | datetime | 是 | 立项时间 |
| updatedAt | 更新时间 | datetime | 是 | 最近更新时间 |
| dueDate | 截止日期 | date | 否 | 目标完成日期 |
| budgetLimit | 预算上限 | number | 否 | 国库预算上限 |
| actualCost | 实际消耗 | number | 否 | 累计实际消耗 |
| tokenCost | token消耗 | number | 否 | 可独立统计 |
| dependencyIds | 依赖项目 | array | 否 | 关联项目 id |
| latestProgress | 最新进展 | text | 否 | 最近状态说明 |
| nextAction | 下一步动作 | text | 否 | 后续动作 |
| attachmentRefs | 附件引用 | array | 否 | 文档、链接等 |
| archivedAt | 归档时间 | datetime | 否 | 归档节点时间 |

| result | 处理结果 | enum | 否 | 通过/退回/驳回/阻塞/完成 |
| comment | 批注 | text | 否 | 处理意见 |
| isTimeout | 是否超时 | boolean | 是 | 默认 false |

## 五、告警字段
| 字段名 | 含义 | 类型 | 必填 | 备注 |
| --- | --- | --- | --- | --- |
| id | 告警主键 | string | 是 | 唯一标识 |
| projectId | 关联项目 | string | 否 | 系统告警可为空 |
| alertType | 告警类型 | enum | 是 | 延期/超预算/卡滞/系统异常等 |
| severity | 严重等级 | enum | 是 | 低/中/高/严重 |
| title | 告警标题 | string | 是 | 简要说明 |
| detail | 告警详情 | text | 否 | 详细说明 |
| status | 告警状态 | enum | 是 | 待处理/处理中/已解决/已忽略 |
| createdAt | 产生时间 | datetime | 是 | 告警创建时间 |
| resolvedAt | 解决时间 | datetime | 否 | 处理完成时间 |
| owner | 处理责任人 | string | 否 | 谁负责关闭 |

## 六、统计字段建议
1. totalProjects：项目总数
2. activeProjects：进行中项目数
3. delayedProjects：延期项目数
4. highRiskProjects：高风险项目数
5. totalBudget：总预算
6. totalActualCost：总实际消耗
7. totalTokenCost：总 token 消耗
8. departmentLoad：部门负载
9. pendingApprovals：待审批事项数
10. blockedProjects：阻塞项目数

## 七、约束说明
1. 阶段、状态、优先级、风险等级必须使用统一枚举值
2. 预算与消耗字段必须统一单位口径
3. 审计字段必须支持关键变更前后留痕
4. 流程记录必须保留时间字段，便于超时与卡滞统计

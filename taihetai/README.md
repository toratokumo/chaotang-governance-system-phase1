# 太和台（朝堂治理系统一期 Demo）

本仓库用于沉淀“朝堂治理系统一期”之制度总纲、页面结构、字段字典、权限矩阵、状态流转与技术落地方案。

## 一期目标

一期不追求装饰性展示，而是优先建立可立项、可流转、可监控、可统计、可留痕、可归档之治理底座。

## 一期范围

纳入范围：
- 项目立项与统一编号
- 项目列表与检索
- 项目详情页
- 基础流程流转
- 朝堂总览页
- 国库统计基础版
- 审计日志页
- 基础告警能力
- 基础角色权限控制

暂不纳入范围：
- 复杂自定义工作流编排
- 全自动 Agent 治理
- 高级报表中心
- 甘特图与复杂时间轴
- 深度外部系统集成
- 纯装饰性展示能力

## 建议实施顺序

1. 先定制度与边界
2. 再定页面与字段
3. 再做数据模型与 API 草案
4. 再开发一期最小可用闭环
5. 最后联调试运行并准备二期增强
